# Code of Conduct

This project follows an **Elegance with Consequences** standard:
contribute with clarity, restraint, and accountability.

## Best practices

- Lead with verified information and say when something is conceptual.
- Preserve the wall-to-proxy-to-spine model in public discussions and
  documentation.
- Keep disagreement technical, specific, and constructive.
- Prefer concise, durable changes over dramatic rewrites.
- Respect the distinction between public framework material and private
  operational internals.

## Unacceptable behavior

- Inventing claims, sources, capabilities, or authority.
- Bypassing review gates for protocol-touching changes.
- Publishing sensitive rules, secrets, or private system logic.
- Using hostile, degrading, or manipulative language toward
  contributors.
- Weakening evidence labels or governance language for marketing.

## Enforcement

Violations may result in comments being removed, issues or pull requests
being closed, or contributors being asked to leave the project. The
Architect (J.G.A.) has final say on project participation.
